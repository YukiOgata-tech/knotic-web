# Knotic 操作マニュアル（契約者向け・たたき台）

最終更新: 2026-02-24  
対象: Knotic 契約者（管理画面を操作する担当者）

## 1. このマニュアルの目的
- 管理画面上でAIボットを作成し、公開・運用するまでの流れをまとめたものです。
- 技術実装（API開発・コード変更）ではなく、画面操作に絞って説明します。

## 2. ログインと初期確認
1. ログインページへアクセスします。
2. メールアドレスとパスワードを入力してログインします。
3. ログイン後、`管理画面（Console）` が表示されます。
4. 画面右上で、ログイン中のメールアドレスと権限（Editor / Reader）を確認します。

## 3. 管理画面メニューの見方
- `Overview`: 全体状況（Bot数、利用状況、契約プラン）
- `Bots`: Botの新規作成、公開切替、公開URL確認
- `Sources`: 学習データ（URL/PDF）登録、インデックス実行
- `API Keys`: 外部連携用APIキーの発行/失効
- `Settings`: AIモデル関連の基本設定

## 4. 基本運用フロー（最重要）
1. `Bots` でBotを作成
2. `Sources` でURLまたはPDFを追加
3. `Sources` でインデックス実行
4. `Bots` で公開設定をON
5. 公開URLや埋め込み設定を確認して運用開始

## 5. Bot作成手順
1. `Bots` を開きます。
2. `新規Bot作成` でBot名を入力します。
3. 必要に応じて説明文を入力し、`Botを作成` を押します。
4. 一覧に新しいBotが追加されれば完了です。

## 6. URLを登録する手順
1. `Sources` を開きます。
2. `URLソース追加` で対象Botを選択します。
3. URLを入力して `URLソースを追加` を押します。
4. 一覧に追加されたことを確認します。
5. `インデックス実行` を押して処理を開始します。

## 7. PDFを登録する手順
1. `Sources` を開きます。
2. `PDFソース追加` で対象Botを選択します。
3. PDFファイルを選択し、`PDFソースを追加` を押します。
4. 一覧に追加されたことを確認します。
5. `インデックス実行` を押して処理を開始します。

## 8. 公開設定と確認
1. `Bots` を開きます。
2. 対象Botの `公開` をONに切り替えます。
3. `Hosted URL` が使えるプランの場合、表示されたURLを確認します。
4. 必要に応じてWidgetトークンを発行し、許可オリジンを設定します。

## 9. APIキーの発行（外部連携時のみ）
1. `API Keys` を開きます。
2. キー名（用途名）を入力し、必要なら有効期限を設定します。
3. `APIキーを発行` を押します。
4. 発行直後に表示されるキーを安全な場所に保管します（再表示されません）。
5. 不要になったキーは `失効` します。

## 10. AI設定の変更
1. `Settings` を開きます。
2. デフォルトモデル、フォールバックモデル、最大出力トークンを設定します。
3. `AI設定を保存` を押します。
4. 反映後、ボット応答品質や速度を確認します。

## 11. 権限の目安
- `Editor`: 作成・更新・公開・設定変更が可能
- `Reader`: 閲覧中心（更新系は不可）

## 12. 運用時のチェックリスト
- 新しい情報を追加したら、インデックス実行を忘れない
- 公開前に対象Botの公開設定を確認
- APIキーは用途ごとに分けて発行
- 不要キーは失効
- 定期的にOverviewで利用状況を確認

## 13. よくあるつまずき
- Botが回答しない:
  - ソース追加後にインデックス実行しているか確認
  - 公開設定がONか確認
- API連携が失敗する:
  - APIキーのコピー漏れ・期限切れ・失効状態を確認
- 期待した情報が回答に出ない:
  - URL/PDFの内容が最新か確認
  - 情報追加後に再インデックス実行
